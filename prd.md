# PRD (Product Requirements Document)

## 1. 문서 정보
- 문서명: 빛공해 감지를 위한 딥러닝 프로젝트 PRD
- 문서 버전: v1.0 (Draft)
- 작성일: 2026-03-06
- 작성 목적: 빛공해 자동 감지/분석 모델 및 운영 시스템 개발 요구사항 정의

---

## 2. 제품 개요

### 2.1 제품명 (가칭)
**LumiWatch AI**

### 2.2 한 줄 설명
도시 및 생활권의 야간 이미지/센서 데이터를 기반으로 빛공해 수준을 자동 감지하고, 원인 광원을 식별하여 개선 우선순위를 제시하는 딥러닝 기반 모니터링 시스템.

### 2.3 배경
- 빛공해는 생태계 교란, 수면장애, 에너지 낭비, 주민 민원 증가를 유발함.
- 기존 측정 방식(현장 계측 중심)은 시간/공간 커버리지가 낮고 운영비가 높음.
- 지자체/연구기관이 상시 모니터링 가능한 데이터 기반 의사결정 도구를 필요로 함.

---

## 3. 문제 정의
1. 현재 빛공해 분석은 수동 조사 비중이 높아 신속 대응이 어렵다.
2. 광원 유형(가로등, 광고판, 건물 외벽조명 등)별 영향 분석이 체계화되어 있지 않다.
3. 지역별/시간대별 빛공해 변화를 정량적으로 추적하기 어렵다.

---

## 4. 목표 및 성공 기준

### 4.1 비즈니스 목표
- 빛공해 민원 대응 시간 단축
- 개선 대상 구역 우선순위 자동 산정
- 야간 조명 정책 수립을 위한 근거 데이터 제공

### 4.2 제품 목표 (MVP)
- 야간 이미지 기반 빛공해 등급 분류(5단계)
- 과도광원 위치 탐지(객체 탐지)
- 구역 단위 빛공해 점수(0~100) 산출

### 4.3 핵심 성과 지표 (KPI)
- 분류 성능: Macro F1 ≥ 0.85
- 광원 탐지 성능: mAP@50 ≥ 0.75
- 점수 예측 성능: MAE ≤ 8.0
- 추론 지연: 단일 이미지 기준 평균 ≤ 300ms (T4급 GPU 환경)
- 운영 안정성: 월간 API 가용성 ≥ 99.5%
